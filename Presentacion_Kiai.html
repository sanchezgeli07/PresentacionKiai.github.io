Ecosistema Actual Kiai

Kiai es una plataforma integral especializada en el cumplimiento normativo del Sector Salud, centralizando la operación financiera y clínica.

Facturación Electrónica

Emisión de facturas, notas crédito/débito y gestión de impuestos. El corazón financiero del sistema.

Módulo Salud (RIPS)

Gestión de pacientes, diagnósticos (CIE) y procedimientos (CUPS). Generación automática de archivos de reporte.

Inventario y Productos

Control de catálogo de servicios médicos y productos, con manejo de unidades de medida.

Clientes y Proveedores

Base de datos centralizada de terceros con información tributaria detallada.

Reportes y Descargas

Descarga masiva de documentos (PDF/XML) y reportes consolidados para auditoría contable.

Configuración y Administración

Personalización visual de empresa, gestión de usuarios, permisos, sucursales y centros de costos.

Mejoras Recientes Implementadas

Hemos optimizado el portal enfocándonos en la integridad de datos y la eficiencia operativa, atacando puntos críticos de dolor del usuario.

  • Blindaje de Datos y Validaciones (Data Integrity)

    Mejora: Implementación de validaciones estrictas en formularios (campos obligatorios, detección de duplicados por documento).

    Impacto: Se eliminan las "fugas de errores" en la aplicación, asegurando la calidad de la información desde el ingreso.

  • Automatización y Eficiencia (Smart Automation)

    Mejora: Operaciones automáticas en campos clave y preselección inteligente de la Resolución de Facturación más usada.

    Impacto: Ahorro significativo de tiempo al emitir facturas o registrar clientes, eliminando la necesidad de ingresar datos repetitivos manualmente.

  • Nuevo Ecosistema de Descarga Masiva

    Mejora: Formulario simplificado para carga de consecutivos con validaciones de Excel (límites de registros), alertas informativas (Spam) y conexión a SharePoint.

    Nuevas Funcionalidades:

    • Gestión de Estados: El cliente puede visualizar en tiempo real el estado de su tarea de descarga masiva.
    • Arquitectura de Jobs (Procesamiento en Segundo Plano):
      • Job 1 (Envío): Consulta y envía periódicamente las tareas de descarga masiva a la API de Kiai.
      • Job 2 (Actualización): Actualiza automáticamente el estado de las tareas en segundo plano.
      • Job 3 (Inserción): Procesa e inserta individualmente las tareas correspondientes a cada documento asociado.

    Impacto: Facilita la gestión masiva de documentos y guía al usuario paso a paso, reduciendo la fricción operativa.

    • Kiai vs. El Mercado (Alegra/Siigo)

    Contexto Competitivo: Las soluciones líderes del mercado como Alegra o Siigo apuntan a un nicho generalista y masivo (Comercio minorista, ferreterías, servicios básicos), donde la prioridad es la simplicidad y la estética. Kiai, en cambio, atiende la complejidad normativa del Sector Salud.

    Hoja de Ruta: El Siguiente Nivel

    Para consolidar a Kiai como la herramienta líder, proponemos las siguientes evoluciones:

    Evolución Tecnológica: Migración a Angular + .NET

    El Gran Salto: Migrar el Frontend actual (Lappiz) a un desarrollo nativo en Angular, conectándolo directamente al Backend existente en .NET mediante la API actual de Kiai.

    Estrategia Técnica:

    • Reutilización: Aprovechar los servicios y lógica de negocio ya estabilizados en el Backend .NET.
    • Independencia: Desacoplar la interfaz gráfica para eliminar conflictos de la plataforma "Low-Code".
    • Escalabilidad: Arquitectura robusta preparada para soportar todos los nuevos desarrollos futuros sin limitaciones.

    Estrategia de Testing (Cypress):

    "En sistemas de misión crítica, la calidad no es negociable. Las pruebas automatizadas son el único mecanismo que garantiza que el software funcione perfectamente antes de llegar al cliente, evitando que sean los usuarios quienes descubran los errores en plena operación."

    • Pruebas E2E: Flujos completos de usuario (login, navegación) en navegador real.
    • Component Testing: Validación aislada de UI y comportamiento de componentes Angular.
    • Integration Tests: Interacción entre componentes y módulos padres/hijos.
    • API / Contract Tests: Verificación de integridad de datos y endpoints sin UI.
    • Visual / Regression: Detección automática de cambios visuales no deseados.
    • Robustez (Red/Edge-cases): Simulación de latencia y fallos de red para validar resiliencia.
    ¿Por qué abandonar el modelo actual (JS en Lappiz)?
    Actual (JS en Contenedor):
    • Conflictos Silenciosos: Al inyectar código JS en una plataforma cerrada, las actualizaciones del proveedor rompen funcionalidades sin aviso.
    • Difícil Mantenimiento: Código disperso en "cajas de texto", sin tipado fuerte ni herramientas de depuración avanzadas.
    • Rendimiento Limitado: Depende del renderizado de la plataforma base.
    • Falta de Garantías (Sin Tests): Inexistencia de pruebas automatizadas de UI, dejando la detección de errores a los usuarios finales en producción.
    • Monolito Rígido: Imposibilidad técnica de implementar arquitectura de Micro Frontends para escalar si el proyecto crece.
    • Dependencia de Proveedor: Tiempos de respuesta dependientes del proveedor ante incidentes críticos para restablecer el servicio.
    • Seguridad Expuesta: Lógica de negocio crítica visible y manipulable en el navegador (Client-Side), aumentando riesgos de seguridad.
    • UX Restringida: Dificultad para crear experiencias de usuario fluidas o personalizadas (por usar Vanilla JS); limitados a lo que la herramienta "Low-Code" permite.
    		 Futuro (Angular + .NET):
    • Estabilidad Total: Código compilado y tipado (TypeScript). Si hay un error, se detecta antes de subir a producción.
    • Arquitectura Profesional: Estructura modular de componentes estándar de la industria.
    • Potencia .NET: Lógica pesada en el servidor, segura y veloz.
    • Calidad Asegurada (Testing): Suite completa (Jasmine/Karma/Cypress) para validar componentes y flujos críticos automáticamente antes de cada entrega.
    • Escalabilidad (Micro Frontends): Capacidad nativa (Module Federation) para dividir la aplicación en micro-apps independientes si el proyecto crece.
    • Control de Infraestructura: Despliegue en nube propia con control total del equipo Kiai, garantizando respuesta inmediata y alta disponibilidad.
    • Seguridad Robusta: Lógica sensible encapsulada en el Backend (.NET), inalcanzable para el usuario final.
    • UX/UI Ilimitada: Libertad total para diseñar interfaces modernas y posibilidad de usar librerías de componentes profesionales.
    Riesgo Operativo (Ambientes):

    Actualmente se desarrolla directo en Producción porque el ambiente de pruebas es inestable y tedioso de actualizar.
    Consecuencia: Cualquier error detiene la operación real del cliente.

    		Ciclo DevOps Seguro:

    Separación estricta: Desarrollo → QA → Producción.
    El código se prueba sin tocar datos reales. Despliegues automáticos (CI/CD) sin intervención manual.

    Desarrollos e Implementaciones Futuras

    Módulos estratégicos proyectados para completar el ecosistema ERP de Salud y cerrar las brechas de mercado.

    1. Inteligencia de Negocios (BI)

    Objetivo: Transformar datos en decisiones estratégicas.

    Objetivo: Visión financiera 360° (P&G).

    Evolucionar de solo facturar (ingresos) a controlar también las cuentas por pagar (egresos). Esto permitirá entregar un Estado de Resultados automático al contador sin salir de Kiai.
